o problema · a tese

    
Velocidade sem método é protótipo disfarçado de produto

    
Documento se ignora, checklist se pula, boa intenção se esquece. Toda prática de engenharia sobe uma escada — e só descansa quando vira mecanismo que acontece sozinho. O trabalho do Renato é empurrar cada regra o mais para a direita possível.

como funciona

    
Ele não escreve código — injeta contexto e cobra método

    
Um servidor MCP + CLI determinística que se conecta ao agente do seu editor. A identidade, os checkpoints e os gates chegam dentro da conversa — ele não precisa abrir arquivo nenhum. E nada sai da sua máquina.

os dois ciclos centrais

    
Test-first e a Segunda Versão, lado a lado

    
Sem teste falhando primeiro, não há implementação. E a primeira versão que funciona ancora o pensamento — por isso a v1 nunca é entregue.

o manifesto

    
Cada frase da casa tem um mecanismo atrás

  

  

    
"Funciona" é opinião. "Esse comando roda e sai isso" é fato.
// critério de aceite como contrato

    
A primeira versão que funciona ancora o pensamento.
// regra da segunda versão

    
O autor jura. O adversário acha.
// revisão adversarial

    
Backup que nunca foi restaurado é fé, não backup.
// verificação de integridade

    
A nota só sobe com mecanismo, nunca com promessa.
// selo da casa (A–F)

    
Se PHP for a melhor saída, PHP vence.
// avaliação, não dogma
